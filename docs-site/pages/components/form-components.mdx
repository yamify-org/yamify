# Form Components

This page documents the form components available in the Yamify application. These components are designed to create consistent, accessible, and user-friendly forms throughout the application.

## FormControl

The FormControl component provides context and accessibility attributes for form elements.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `id` | `string` | auto-generated | ID for the form control |
| `isRequired` | `boolean` | `false` | Whether the form control is required |
| `isInvalid` | `boolean` | `false` | Whether the form control has an error |
| `isDisabled` | `boolean` | `false` | Whether the form control is disabled |
| `isReadOnly` | `boolean` | `false` | Whether the form control is read-only |
| `label` | `string` | `undefined` | Label text for the form control |
| `helperText` | `ReactNode` | `undefined` | Helper text displayed below the form control |
| `errorText` | `ReactNode` | `undefined` | Error message displayed when isInvalid is true |

### Examples

```tsx
// Basic usage
<FormControl id="email" label="Email Address" isRequired>
  <Input type="email" placeholder="Enter your email" />
</FormControl>

// With helper text
<FormControl 
  id="password" 
  label="Password" 
  helperText="Password must be at least 8 characters long."
>
  <Input type="password" />
</FormControl>

// With error state
<FormControl 
  id="username" 
  label="Username" 
  isInvalid={!!errors.username}
  errorText={errors.username}
>
  <Input value={username} onChange={(e) => setUsername(e.target.value)} />
</FormControl>
```

## FormLabel

The FormLabel component renders a label for form controls.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `htmlFor` | `string` | `undefined` | ID of the form element the label is for |
| `isRequired` | `boolean` | `false` | Whether to show a required indicator |

### Examples

```tsx

// Required field

```

## Input

The Input component renders an HTML input element with Yamify styling.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `id` | `string` | `undefined` | Unique ID for the input |

| `value` | `string | number` | `undefined` | Current value of the input |
| `defaultValue` | `string | number` | `undefined` | Default value of the input |
| `placeholder` | `string` | `''` | Placeholder text |
| `onChange` | `(e: ChangeEvent<HTMLInputElement>) => void` | `undefined` | Function called when the input value changes |
| `onBlur` | `(e: FocusEvent<HTMLInputElement>) => void` | `undefined` | Function called when the input loses focus |
| `onFocus` | `(e: FocusEvent<HTMLInputElement>) => void` | `undefined` | Function called when the input gains focus |
| `isDisabled` | `boolean` | `false` | Whether the input is disabled |
| `isReadOnly` | `boolean` | `false` | Whether the input is read-only |
| `isRequired` | `boolean` | `false` | Whether the input is required |
| `isInvalid` | `boolean` | `false` | Whether the input has an error |
| `autoComplete` | `string` | `undefined` | Autocomplete attribute |
| `autoFocus` | `boolean` | `false` | Whether the input should be focused automatically |
| `size` | `'sm' | 'md' | 'lg'` | `'md'` | Size of the input |
| `leftIcon` | `ReactNode` | `undefined` | Icon to display on the left side of the input |
| `rightIcon` | `ReactNode` | `undefined` | Icon to display on the right side of the input |

### Examples

```tsx
// Basic text input
<Input 
  id="firstName"
  name="firstName"
  placeholder="Enter your first name"
  value={firstName}
  onChange={(e) => setFirstName(e.target.value)}

// Email input with validation
<Input

  name="email"

// Password input with icon

  type="password"

  onChange={handlePasswordChange}
  rightIcon={
    <button onClick={togglePasswordVisibility}>
      {showPassword ? <EyeOffIcon /> : <EyeIcon />}
    </button>
  }
/>

// Disabled input
<Input 
  value="Read-only value"
  isDisabled
/>

// Input with left icon
<Input 
  placeholder="Search..."
  leftIcon={<SearchIcon />}
/>

// Input with left icon
<Input 
  placeholder="Search..."
  leftIcon={<SearchIcon />}
/>

// Disabled input

  value="Read-only value"
  isDisabled
/>
```

## Form Validation

Yamify's form components are designed to work with various form validation libraries. Here's an example using React Hook Form:

```tsx
import { useForm } from 'react-hook-form';

function SignupForm() {
  const { 
    register, 
    handleSubmit, 
    formState: { errors, isSubmitting } 
  } = useForm();

  const onSubmit = async (data) => {
    // Submit form data
    console.log(data);
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <Stack spacing={4}>
        <FormControl isInvalid={!!errors.name}>
          <FormLabel>Full Name</FormLabel>
          <Input
            {...register('name', {
              required: 'Name is required',
              minLength: { value: 2, message: 'Name must be at least 2 characters' }
            })}
          />
          {errors.name && (
            <div className="text-red-500 text-sm mt-1">{errors.name.message}</div>
          )}
        </FormControl>

        <FormControl isInvalid={!!errors.email}>
          <FormLabel>Email Address</FormLabel>
          <Input
            type="email"
            {...register('email', {
              required: 'Email is required',
              pattern: {
                value: /^[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,}$/i,
                message: 'Invalid email address'
              }
            })}
          />
          {errors.email && (
            <div className="text-red-500 text-sm mt-1">{errors.email.message}</div>
          )}
        </FormControl>

        <FormControl isInvalid={!!errors.password}>
          <FormLabel>Password</FormLabel>
          <Input
            type="password"
            {...register('password', {
              required: 'Password is required',
              minLength: {
                value: 8,
                message: 'Password must be at least 8 characters'
              }
            })}
          />
          {errors.password && (
            <div className="text-red-500 text-sm mt-1">{errors.password.message}</div>
          )}
        </FormControl>

        <FormControl isInvalid={!!errors.terms}>
          <Checkbox
            {...register('terms', {
              required: 'You must agree to the terms'
            })}
          >
            I agree to the terms and conditions
          </Checkbox>
          {errors.terms && (
            <div className="text-red-500 text-sm mt-1">{errors.terms.message}</div>
          )}
        </FormControl>

        <Button type="submit" isLoading={isSubmitting} isFullWidth>
          Sign Up
        </Button>
      </Stack>
    </form>
  );
}
```

## Building Complex Forms

```tsx
    e.preventDefault();
    setIsLoading(true);
    
    try {
      // Form submission logic
      await onSubmit(formData);
      toast.success('Profile updated successfully');
    } catch (error) {
      toast.error('Failed to update profile');

      setIsLoading(false);
    }

  return (
    <form onSubmit={handleSubmit}>
      <Stack spacing={6}>
        {/* Personal Information Section */}
```

## Accessibility Considerations

All form components in Yamify are built with accessibility in mind:

- All form controls have proper labeling and ARIA attributes
- Error states are properly communicated to screen readers
- Form controls can be navigated and operated using only the keyboard
- Focus states are clearly visible
- Color contrast meets WCAG 2.1 AA standards

## Best Practices

1. **Always use labels**: Every form control should have a visible label that clearly describes its purpose.

2. **Provide helpful error messages**: Error messages should be clear, specific, and suggest how to fix the issue.

3. **Group related fields**: Use `<fieldset>` and `<legend>` (via FormControl) to group related form controls.

4. **Show validation in real-time**: Validate fields as users complete them rather than waiting for form submission.

5. **Use appropriate input types**: Use specialized input types like `email`, `tel`, and `number` to provide the best user experience.

6. **Maintain a consistent layout**: Keep form layouts consistent across your application.

7. **Mark required fields**: Clearly indicate which fields are required using the `isRequired` prop.

8. **Provide context**: Use helper text to provide additional information or instructions.

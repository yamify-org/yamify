# UI Components

This page documents the reusable UI components available in the Yamify application. These components serve as the building blocks for the user interface and help maintain consistency throughout the application.

## Button

The Button component is used for actions and navigation.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'primary' | 'secondary' | 'tertiary' | 'danger'` | `'primary'` | The visual style of the button |
| `size` | `'sm' | 'md' | 'lg'` | `'md'` | The size of the button |
| `isFullWidth` | `boolean` | `false` | Whether the button should take up the full width of its container |
| `isDisabled` | `boolean` | `false` | Whether the button is disabled |
| `isLoading` | `boolean` | `false` | Whether the button is in a loading state |
| `leftIcon` | `ReactNode` | `undefined` | Icon to display on the left side of the button text |
| `rightIcon` | `ReactNode` | `undefined` | Icon to display on the right side of the button text |
| `onClick` | `() => void` | `undefined` | Function to call when the button is clicked |

### Examples

```tsx
// Basic usage
<Button variant="primary" onClick={() => console.log('Clicked!')}>Click Me</Button>

// With icons
<Button leftIcon={<IconMail />} variant="secondary">Send Email</Button>

// Loading state
<Button isLoading>Processing</Button>

// Full width
<Button isFullWidth variant="tertiary">Full Width Button</Button>
```

## Card

The Card component is used to group related information and actions.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'elevated' | 'outlined'` | `'elevated'` | The visual style of the card |
| `isHoverable` | `boolean` | `false` | Whether the card should have a hover effect |
| `isClickable` | `boolean` | `false` | Whether the card should have a pointer cursor and click effect |
| `onClick` | `() => void` | `undefined` | Function to call when the card is clicked |

### Examples

```tsx
// Basic usage
<Card>
  <CardHeader>
    <CardTitle>Card Title</CardTitle>
    <CardDescription>This is a description of the card.</CardDescription>
  </CardHeader>
  <CardBody>
    <p>Card content goes here.</p>
  </CardBody>
  <CardFooter>
    <Button>Action</Button>
  </CardFooter>
</Card>

// Outlined variant
<Card variant="outlined" isHoverable>
  <CardBody>
    <p>This is an outlined card with hover effect.</p>
  </CardBody>
</Card>

// Clickable card
<Card isClickable onClick={() => console.log('Card clicked')}>
  <CardBody>
    <p>Click this card!</p>
  </CardBody>
</Card>
```

## Input

The Input component is used for text input fields.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `id` | `string` | Required | Unique identifier for the input |
| `label` | `string` | `undefined` | Label text for the input |
| `placeholder` | `string` | `''` | Placeholder text |
| `value` | `string` | `''` | Current value of the input |
| `onChange` | `(event: ChangeEvent<HTMLInputElement>) => void` | Required | Function to call when the input value changes |
| `type` | `'text' | 'email' | 'password' | 'number' | 'tel'` | `'text'` | Type of input |
| `isDisabled` | `boolean` | `false` | Whether the input is disabled |
| `isReadOnly` | `boolean` | `false` | Whether the input is read-only |
| `isRequired` | `boolean` | `false` | Whether the input is required |
| `isInvalid` | `boolean` | `false` | Whether the input has an error |
| `errorMessage` | `string` | `undefined` | Error message to display when isInvalid is true |
| `helperText` | `string` | `undefined` | Helper text to display below the input |
| `leftIcon` | `ReactNode` | `undefined` | Icon to display on the left side of the input |
| `rightIcon` | `ReactNode` | `undefined` | Icon to display on the right side of the input |

### Examples

```tsx
// Basic usage
<Input
  id="email"
  label="Email Address"
  type="email"
  placeholder="Enter your email"
  value={email}
  onChange={(e) => setEmail(e.target.value)}
/>

// With error state
<Input
  id="password"
  label="Password"
  type="password"
  value={password}
  onChange={(e) => setPassword(e.target.value)}
  isInvalid={!!passwordError}
  errorMessage={passwordError}
/>

// With icons
<Input
  id="search"
  placeholder="Search..."
  value={searchTerm}
  onChange={(e) => setSearchTerm(e.target.value)}
  leftIcon={<IconSearch />}
  rightIcon={searchTerm ? <IconX onClick={() => setSearchTerm('')} /> : undefined}
/>
```

## Alert

The Alert component is used to display important messages to the user.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'info' | 'success' | 'warning' | 'error'` | `'info'` | The visual style of the alert |
| `title` | `string` | `undefined` | Title of the alert |
| `icon` | `ReactNode` | `undefined` | Custom icon to display |
| `isDismissable` | `boolean` | `false` | Whether the alert can be dismissed |
| `onDismiss` | `() => void` | `undefined` | Function to call when the alert is dismissed |

### Examples

```tsx
// Basic usage
<Alert variant="info">
  <p>This is an informational alert.</p>
</Alert>

// With title and dismiss button
<Alert 
  variant="success" 
  title="Operation Successful" 
  isDismissable 
  onDismiss={() => setShowAlert(false)}
>
  <p>Your changes have been saved successfully.</p>
</Alert>

// Warning alert
<Alert variant="warning" title="Attention Required">
  <p>Your subscription will expire in 3 days. Please renew to avoid service interruption.</p>
</Alert>

// Error alert
<Alert variant="error" title="Error Occurred">
  <p>Unable to save changes. Please try again later.</p>
</Alert>
```

## Dropdown

The Dropdown component provides a menu of options that can be triggered by a button.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `trigger` | `ReactNode` | Required | Element that triggers the dropdown |
| `placement` | `'top' | 'top-start' | 'top-end' | 'bottom' | 'bottom-start' | 'bottom-end' | 'left' | 'right'` | `'bottom-start'` | Placement of the dropdown relative to the trigger |
| `isOpen` | `boolean` | `undefined` | Controlled state of the dropdown |
| `onOpen` | `() => void` | `undefined` | Function called when the dropdown opens |
| `onClose` | `() => void` | `undefined` | Function called when the dropdown closes |

### Examples

```tsx
// Basic usage
<Dropdown trigger={<Button>Options</Button>}>
  <DropdownItem onClick={() => console.log('Edit')}>Edit</DropdownItem>
  <DropdownItem onClick={() => console.log('Duplicate')}>Duplicate</DropdownItem>
  <DropdownItem onClick={() => console.log('Delete')}>Delete</DropdownItem>
</Dropdown>

// With divider and disabled item
<Dropdown trigger={<Button>Actions</Button>}>
  <DropdownItem icon={<IconEdit />} onClick={() => console.log('Edit')}>Edit</DropdownItem>
  <DropdownItem icon={<IconCopy />} onClick={() => console.log('Duplicate')}>Duplicate</DropdownItem>
  <DropdownDivider />
  <DropdownItem 
    icon={<IconTrash />} 
    onClick={() => console.log('Delete')} 
    isDisabled={!canDelete}
  >
    Delete
  </DropdownItem>
</Dropdown>

// Controlled dropdown
<Dropdown 
  trigger={<Button>User Settings</Button>}
  isOpen={isDropdownOpen}
  onOpen={() => setIsDropdownOpen(true)}
  onClose={() => setIsDropdownOpen(false)}
>
  <DropdownItem>Profile</DropdownItem>
  <DropdownItem>Settings</DropdownItem>
  <DropdownItem>Logout</DropdownItem>
</Dropdown>
```

## Modal

The Modal component displays content in a layer that sits above the page.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `isOpen` | `boolean` | Required | Whether the modal is visible |
| `onClose` | `() => void` | Required | Function called when the modal should close |
| `title` | `string` | `undefined` | Title of the modal |
| `size` | `'sm' | 'md' | 'lg' | 'xl' | 'full'` | `'md'` | Size of the modal |
| `closeOnOverlayClick` | `boolean` | `true` | Whether clicking the overlay should close the modal |
| `closeOnEsc` | `boolean` | `true` | Whether pressing the Escape key should close the modal |
| `isCentered` | `boolean` | `true` | Whether the modal should be centered vertically |

### Examples

```tsx
// Basic usage
<Button onClick={() => setIsModalOpen(true)}>Open Modal</Button>

<Modal isOpen={isModalOpen} onClose={() => setIsModalOpen(false)} title="Modal Title">
  <p>This is the modal content.</p>
  
  <div className="flex justify-end gap-2 mt-4">
    <Button variant="tertiary" onClick={() => setIsModalOpen(false)}>Cancel</Button>
    <Button>Confirm</Button>
  </div>
</Modal>

// Custom size and behavior
<Modal 
  isOpen={isCustomModalOpen} 
  onClose={() => setIsCustomModalOpen(false)} 
  title="Large Modal"
  size="lg"
  closeOnOverlayClick={false}
>
  <p>This is a large modal that can't be closed by clicking the overlay.</p>
  <Button onClick={() => setIsCustomModalOpen(false)}>Close</Button>
</Modal>
```

## Tabs

The Tabs component allows users to switch between different views within the same context.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultTab` | `string` | `undefined` | The ID of the tab to select by default |
| `selectedTab` | `string` | `undefined` | Controlled selected tab ID |
| `onChange` | `(tabId: string) => void` | `undefined` | Function called when a tab is selected |
| `variant` | `'line' | 'enclosed' | 'pills'` | `'line'` | Visual style of the tabs |

### Examples

```tsx
// Basic usage
<Tabs defaultTab="details">
  <TabList>
    <Tab id="details">Details</Tab>
    <Tab id="settings">Settings</Tab>
    <Tab id="history">History</Tab>
  </TabList>
  
  <TabPanels>
    <TabPanel id="details">
      <p>Details content here</p>
    </TabPanel>
    <TabPanel id="settings">
      <p>Settings content here</p>
    </TabPanel>
    <TabPanel id="history">
      <p>History content here</p>
    </TabPanel>
  </TabPanels>
</Tabs>

// Controlled tabs
<Tabs 
  selectedTab={selectedTab} 
  onChange={setSelectedTab}
  variant="pills"
>
  <TabList>
    <Tab id="tab1">Tab 1</Tab>
    <Tab id="tab2">Tab 2</Tab>
    <Tab id="tab3" isDisabled>Tab 3 (Disabled)</Tab>
  </TabList>
  
  <TabPanels>
    <TabPanel id="tab1">
      <p>Content for Tab 1</p>
    </TabPanel>
    <TabPanel id="tab2">
      <p>Content for Tab 2</p>
    </TabPanel>
    <TabPanel id="tab3">
      <p>Content for Tab 3</p>
    </TabPanel>
  </TabPanels>
</Tabs>
```

## Toast

The Toast component is used to show brief notifications to users.

Yamify uses React Toastify for toast notifications. Here's how to use it:

### Examples

```tsx
import { toast } from 'react-toastify';

// Basic usage
const showToast = () => {
  toast('This is a toast notification');
};

// Different toast types
const showSuccessToast = () => {
  toast.success('Operation completed successfully!');
};

const showErrorToast = () => {
  toast.error('An error occurred');
};

const showInfoToast = () => {
  toast.info('Just so you know...');
};

const showWarningToast = () => {
  toast.warning('This is a warning');
};

// Custom toast content
const showCustomToast = () => {
  toast(
    <div>
      <h4 className="font-bold">Custom Toast</h4>
      <p>This is a custom toast with more complex content.</p>
      <Button size="sm" className="mt-2">Action</Button>
    </div>
  );
};

// With options
const showToastWithOptions = () => {
  toast.success('Changes saved', {
    position: 'bottom-right',
    autoClose: 5000,
    hideProgressBar: false,
    closeOnClick: true,
    pauseOnHover: true,
    draggable: true
  });
};
```

## Accessibility Considerations

All UI components in Yamify are designed with accessibility in mind:

- Buttons and interactive elements have appropriate ARIA attributes
- Focus management is handled for modal dialogs and dropdowns
- Color contrast meets WCAG 2.1 AA standards
- Keyboard navigation is supported for all interactive components
- Screen reader announcements are provided for dynamic content changes

## Using UI Components Together

Yamify's UI components are designed to work together seamlessly. Here's an example of combining multiple components to create a user interface:

```tsx
<Card>
  <CardHeader>
    <CardTitle>User Profile</CardTitle>
  </CardHeader>
  
  <CardBody>
    <Tabs defaultTab="info">
      <TabList>
        <Tab id="info">Personal Info</Tab>
        <Tab id="security">Security</Tab>
      </TabList>
      
      <TabPanels>
        <TabPanel id="info">
          <div className="space-y-4">
            <Input
              id="name"
              label="Full Name"
              value={name}
              onChange={(e) => setName(e.target.value)}
            />
            
            <Input
              id="email"
              type="email"
              label="Email Address"
              value={email}
              onChange={(e) => setEmail(e.target.value)}
            />
            
            {saveError && (
              <Alert variant="error">
                {saveError}
              </Alert>
            )}
          </div>
        </TabPanel>
        
        <TabPanel id="security">
          <div className="space-y-4">
            <Alert variant="info">
              <p>Password must be at least 8 characters long and include a number.</p>
            </Alert>
            
            <Input
              id="current-password"
              type="password"
              label="Current Password"
              value={currentPassword}
              onChange={(e) => setCurrentPassword(e.target.value)}
            />
            
            <Input
              id="new-password"
              type="password"
              label="New Password"
              value={newPassword}
              onChange={(e) => setNewPassword(e.target.value)}
            />
          </div>
        </TabPanel>
      </TabPanels>
    </Tabs>
  </CardBody>
  
  <CardFooter>
    <div className="flex justify-end gap-2">
      <Button variant="tertiary">Cancel</Button>
      <Button isLoading={isSaving} onClick={handleSave}>Save Changes</Button>
    </div>
  </CardFooter>
</Card>
```

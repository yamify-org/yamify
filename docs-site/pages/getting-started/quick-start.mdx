# Quick Start Guide

This guide will help you get Yamify up and running quickly so you can start exploring its features. For a more detailed setup, refer to the [Installation Guide](/getting-started/installation).

## Prerequisites

Before you begin, ensure you have the following installed:

- Node.js (v18.0.0 or later)
- npm (v8.0.0 or later) or yarn (v1.22.0 or later)
- Git

## Step 1: Clone the Repository

```bash
git clone https://github.com/yamify-org/yamify.git
cd yamify
```

## Step 2: Install Dependencies

```bash
npm install
# or
yarn install
```

## Step 3: Run the Development Server

```bash
npm run dev
# or
yarn dev
```

The application will be available at [http://localhost:3000](http://localhost:3000).

## Step 4: Explore the Application

You can now explore the Yamify application. Here are some key areas to check out:

- **Home Page**: The landing page of the application
- **Dashboard**: The main interface (accessible at `/dashboard`)
- **Authentication**: Login and registration features (accessible at `/auth`)

## Next Steps

Now that you have Yamify running locally, here are some suggested next steps:

### Customize the Application

To customize Yamify for your needs:

1. Modify the configuration in `next.config.ts`
2. Update the styles in `src/app/globals.css`
3. Modify components in the `src/components` directory

### Learn More About the Project Structure

To understand how the project is organized, check out:

- [Project Structure](/architecture/project-structure)
- [Technologies](/architecture/technologies)
- [Component Documentation](/components/overview)

### Set Up for Production

When you're ready to deploy Yamify to production:

```bash
npm run build
npm run start
# or
yarn build
yarn start
```

For detailed deployment instructions, see the [Installation Guide - Production Deployment](/getting-started/installation#building-for-production) section.

## Troubleshooting

If you encounter any issues during setup:

1. Check that your Node.js version meets the requirements
2. Ensure all dependencies were installed correctly
3. Check for any error messages in the console
4. Consult the [Troubleshooting](/support/troubleshooting) page

If you need further assistance, [contact the support team](/support/contact).

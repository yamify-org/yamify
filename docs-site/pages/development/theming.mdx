# Theming and Customization

Yamify provides a flexible theming system that allows you to customize the appearance of your application to match your brand identity. This guide explains how to use and extend Yamify's theme system.

## Theme Architecture

Yamify's theming is built on a combination of TailwindCSS and CSS variables. The system provides:

- Light and dark mode support out-of-the-box
- Customizable color palettes
- Typography control
- Spacing and layout scales
- Component-specific theming

## Customizing the Theme

### Creating a Custom Theme

To create a custom theme, you'll need to extend the TailwindCSS configuration and override CSS variables.

1. Create a `theme.css` file in your styles directory:

```css
:root {
  /* Primary colors */
  --color-primary-50: #f0f9ff;
  --color-primary-100: #e0f2fe;
  --color-primary-200: #bae6fd;
  --color-primary-300: #7dd3fc;
  --color-primary-400: #38bdf8;
  --color-primary-500: #0ea5e9;
  --color-primary-600: #0284c7;
  --color-primary-700: #0369a1;
  --color-primary-800: #075985;
  --color-primary-900: #0c4a6e;
  --color-primary-950: #082f49;
  
  /* Other theme colors... */
}

/* Dark mode overrides */
.dark {
  --color-primary-50: #082f49;
  --color-primary-100: #0c4a6e;
  --color-primary-200: #075985;
  --color-primary-300: #0369a1;
  --color-primary-400: #0284c7;
  --color-primary-500: #0ea5e9;
  --color-primary-600: #38bdf8;
  --color-primary-700: #7dd3fc;
  --color-primary-800: #bae6fd;
  --color-primary-900: #e0f2fe;
  --color-primary-950: #f0f9ff;
  
  /* Other dark mode colors... */
}
```

2. Extend your TailwindCSS configuration:

```js
// tailwind.config.js
const { colors } = require('./src/theme/colors');

module.exports = {
  theme: {
    extend: {
      colors,
      // Other theme extensions
    },
  },
  // Rest of your Tailwind config
};
```

## Theme Provider

Yamify provides a `ThemeProvider` component that you can use to set theme preferences and manage theme changes:

```tsx
// _app.tsx or layout.tsx
import { ThemeProvider } from '@yamify/core';

export default function App({ Component, pageProps }) {
  return (
    <ThemeProvider defaultTheme="light" storageKey="yamify-theme-preference">
      <Component {...pageProps} />
    </ThemeProvider>
  );
}
```

### Theme Provider Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultTheme` | `'light' \| 'dark' \| 'system'` | `'system'` | The default theme to use |
| `storageKey` | `string` | `'theme-preference'` | Local storage key for saving theme preference |
| `themes` | `string[]` | `['light', 'dark']` | Available themes |
| `enableSystem` | `boolean` | `true` | Whether to allow system preference |

## Using the Theme Hook

Yamify provides a `useTheme` hook to interact with the theme system:

```tsx
import { useTheme } from '@yamify/core';

function ThemeToggle() {
  const { theme, setTheme } = useTheme();
  
  return (
    <div>
      <p>Current theme: {theme}</p>
      <button onClick={() => setTheme('light')}>Light</button>
      <button onClick={() => setTheme('dark')}>Dark</button>
      <button onClick={() => setTheme('system')}>System</button>
    </div>
  );
}
```

## Creating a Theme Toggle Component

Here's a complete example of a theme toggle component:

```tsx
import { useTheme } from '@yamify/core';
import { MoonIcon, SunIcon, ComputerDesktopIcon } from '@heroicons/react/24/outline';

export function ThemeToggle() {
  const { theme, setTheme } = useTheme();

  return (
    <div className="flex items-center space-x-2 rounded-md border p-2">
      <button
        className={`rounded-md p-2 ${theme === 'light' ? 'bg-primary-200' : ''}`}
        onClick={() => setTheme('light')}
        aria-label="Light mode"
      >
        <SunIcon className="h-5 w-5" />
      </button>
      <button
        className={`rounded-md p-2 ${theme === 'dark' ? 'bg-primary-200' : ''}`}
        onClick={() => setTheme('dark')}
        aria-label="Dark mode"
      >
        <MoonIcon className="h-5 w-5" />
      </button>
      <button
        className={`rounded-md p-2 ${theme === 'system' ? 'bg-primary-200' : ''}`}
        onClick={() => setTheme('system')}
        aria-label="System preference"
      >
        <ComputerDesktopIcon className="h-5 w-5" />
      </button>
    </div>
  );
}
```

## Advanced Customization

### Custom Color Schemes

You can create custom color schemes by extending the theme configuration:

```js
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        'brand': {
          50: '#f9fafb',
          100: '#f3f4f6',
          // ... more shades
          900: '#111827',
        },
        // You can then use this in your components: text-brand-500, bg-brand-100, etc.
      },
    },
  },
};
```

### Component Style Overrides

You can override specific component styles by creating a component style map:

```tsx
// theme/componentStyles.ts
export const componentStyles = {
  Button: {
    baseStyle: {
      fontWeight: 'semibold',
      borderRadius: 'lg',
    },
    variants: {
      custom: {
        bg: 'brand.500',
        color: 'white',
        _hover: {
          bg: 'brand.600',
        },
      },
    },
  },
  // Other component overrides
};

// Then import and pass to ThemeProvider
<ThemeProvider componentStyles={componentStyles}>
  {/* Your app */}
</ThemeProvider>
```

## Runtime Theme Switching

Yamify includes utility functions for changing themes at runtime without page reloads:

```tsx
import { applyTheme } from '@yamify/core';

// Apply a theme at runtime
function applyCustomTheme() {
  const customTheme = {
    primary: '#ff0000', // Red primary color
    // Other theme values
  };
  
  applyTheme('custom-theme', customTheme);
}
```

## Accessibility Considerations

When customizing your theme, ensure you maintain good color contrast ratios for accessibility:

- Text on backgrounds should have a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text
- Use the WebAIM contrast checker to verify your color combinations
- Ensure focus states are clearly visible
- Provide sufficient color variations for different states (hover, active, disabled)

## Best Practices

- **Consistency**: Maintain consistency by using theme values throughout your app
- **Design Tokens**: Use design tokens rather than hard-coded values
- **Dark Mode**: Always test your theme in both light and dark modes
- **Performance**: Minimize runtime theme changes that could cause layout shifts
- **Responsiveness**: Ensure your theme works across different screen sizes

## Examples

### Complete Theme Example

Here's a complete example of a custom Yamify theme:

```tsx
// theme/index.ts
import { colors } from './colors';
import { typography } from './typography';
import { componentStyles } from './componentStyles';

export const theme = {
  colors,
  typography,
  componentStyles,
  shadows: {
    sm: '0 1px 2px 0 rgba(0, 0, 0, 0.05)',
    md: '0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -1px rgba(0, 0, 0, 0.06)',
    lg: '0 10px 15px -3px rgba(0, 0, 0, 0.1), 0 4px 6px -2px rgba(0, 0, 0, 0.05)',
  },
  spacing: {
    // Custom spacing scale if needed
  },
  borderRadius: {
    none: '0',
    sm: '0.125rem',
    md: '0.375rem',
    lg: '0.5rem',
    xl: '0.75rem',
    '2xl': '1rem',
    full: '9999px',
  },
};

// In your _app.tsx
import { ThemeProvider } from '@yamify/core';
import { theme } from './theme';

export default function App({ Component, pageProps }) {
  return (
    <ThemeProvider theme={theme}>
      <Component {...pageProps} />
    </ThemeProvider>
  );
}
```
